# Module 0 — Mise en route

> **Bienvenue !** Ce module est ton point de départ. À la fin, tu auras un ordinateur prêt à coder avec Claude Code, un compte GitHub fonctionnel, et tu auras déjà eu ta première vraie conversation avec un assistant IA dans ton terminal.

## À qui s'adresse ce module

Ce module est conçu pour toi si :

- tu n'as **jamais ouvert un terminal** (ou tu l'as fait une fois, par accident, et tu l'as vite refermé) ;
- tu entends parler de Claude Code, GitHub, Git, et tu te demandes comment t'y mettre concrètement ;
- tu veux apprendre proprement, sans sauter d'étape, pour ne pas être perdu plus tard.

Si tu es déjà développeur expérimenté, tu peux survoler ce module, mais lis quand même les sections sur l'installation de Claude Code et les bonnes pratiques de configuration — il y a probablement des choses utiles.

## Ce que tu vas savoir faire à la fin

- [ ] Ouvrir et utiliser un terminal sur ton système
- [ ] Naviguer dans tes fichiers en ligne de commande
- [ ] Avoir un compte GitHub sécurisé avec la double authentification
- [ ] Avoir un compte Anthropic avec le bon plan pour suivre la formation
- [ ] Installer Git, VS Code et Claude Code sur ton ordinateur
- [ ] Connecter ton ordinateur à GitHub (sans devoir taper ton mot de passe à chaque fois)
- [ ] Lancer Claude Code et lui faire effectuer une vraie tâche
- [ ] Créer ton premier repo, le pousser sur GitHub, et fêter ça

## Durée estimée

**2h30 à 4h selon ton rythme.** Prends une demi-journée tranquille. Ne te précipite pas. Ce module est la base de tout ce qui suit — si tu le fais bien, le reste de la formation sera infiniment plus simple.

## Prérequis

- Un ordinateur sous **macOS, Windows (10 ou plus récent), ou Linux**.
- Une connexion internet stable.
- Une adresse email à laquelle tu as accès.
- Une carte bancaire (pour l'abonnement Anthropic — Claude Pro suffit à ce stade).
- De la patience et un café.

## Comment utiliser ce repo

Ce repo contient :

- `chapitres/` : les leçons à lire dans l'ordre.
- `exercices/` : les fichiers de départ pour les exercices pratiques.
- `assets/` : images et ressources visuelles.

Tu vas cloner ce repo sur ton ordinateur au cours du chapitre 0.5. Ne le télécharge pas en ZIP — c'est justement l'occasion d'apprendre `git clone`.

## Si tu es bloqué·e

Trois réflexes, dans cet ordre :

1. **Relis le chapitre en cours.** 90% des blocages viennent d'une étape sautée.
2. **Consulte la section "Pièges classiques"** à la fin de chaque chapitre.
3. **Demande de l'aide sur le Discord de la formation.** Tu n'es pas seul·e, et la question que tu te poses, quelqu'un d'autre se la pose en ce moment même.
